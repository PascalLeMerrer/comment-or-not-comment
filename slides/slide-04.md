## Pourquoi commenter ?
selon Antirez

1. Il manque souvent le pourquoi
2. Réduire la charge cognitive

Note:

De nombreux développeurs pensent qu'un code bien conçu peut se passer de commentaires.
Antirez n'est pas de cet avis, pour deux raisons principales.

Même si le code est clair, il manque souvent une info essentielle :
pourquoi avoir choisi cette façon de coder plutôt qu'une autre,
qui pourrait paraitre plus naturelle au lecteur.

Et pui sil veut faciliter la compréhension du code par le lecteur.
